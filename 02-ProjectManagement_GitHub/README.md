# 02 - Project Management with GitHub Projects
> **Learning Path:** [Manage your work with GitHub Projects](https://learn.microsoft.com/en-us/training/modules/manage-work-github-projects/)

## PBL Objective: The "Self-Automating Student"
The goal of this project is not just to finish the module, but to build a **GitHub Project (v2)** that manages my own learning journey. I am applying **Everything as Code** by treating my study notes as versioned artifacts and my progress as trackable data.

## 🛠 Methodology & Workflow
To simulate a professional Platform Engineering environment, this project adheres to:
* **Trunk-Based Development (TBD):** Short-lived feature branches merged frequently to `main`.
* **Atomic Commits:** Each commit represents a single, logical unit of change.
* **Everything as Code (EaC):**
    * **Document as Code (DaC):** Using Markdown for structured, versioned documentation.
    * **Knowledge as Code (KaC):** Using `metadata.yaml` for data-driven progress tracking.

---

## Project Architecture
The project is structured into three primary components:
1.  **The Backlog:** GitHub Issues representing each Microsoft Learning Unit.
2.  **The Automation:** GitHub Project workflows to move items from `Todo` to `Done`.
3.  **The Repository:** This folder acting as the "Source of Truth" for theory and labs.
